# Identidade Visual — Paulo Design

## Essência

Clareza operacional, execução confiável e sofisticação discreta.

A interface deve parecer feita para decisão real, não para impressionar em pitch. O usuário deve sentir que o produto reduz ruído, organiza trabalho e entrega prova concreta.

## Personalidade visual

- **Direta:** hierarquia clara, sem enfeite inútil.
- **Confiável:** contraste alto, estados explícitos, feedback visível.
- **Premium discreta:** detalhes refinados, sem luxo teatral.
- **Operacional:** componentes densos o suficiente para trabalho real.
- **Humana:** cantos suaves, microcopy natural, nada robótico.

## Paleta

- Fundo: `#F7F5EF`
- Superfície: `#FFFFFF`
- Texto: `#171A1C`
- Texto secundário: `#5D646B`
- Primária: `#123A2F`
- Acento: `#B28A3C`
- Borda: `#DDD5C8`

Use verde como direção/ação e dourado como destaque executivo. Não use dourado como cor dominante.

## Tipografia

- Interface: Inter ou system-ui.
- Destaques editoriais: Fraunces ou serif equivalente.
- Código/dados: JetBrains Mono ou monospace equivalente.

## Layout

- Espaço generoso, mas sem landing page vazia.
- Cards com bordas visíveis e sombra leve.
- Informação importante acima da dobra.
- Estados vazios devem orientar próxima ação.
- Tabelas e painéis precisam priorizar leitura e decisão.

## Copy

Bom:

- “Pronto para revisar”
- “Pendência sem documento fiscal”
- “Resultado consolidado”
- “Última validação: hoje, 08:12”

Ruim:

- “Solução inovadora movida por IA”
- “Potencialize seus resultados”
- “Experiência revolucionária”
- “Clique aqui para começar sua jornada”

## Critérios de qualidade

Antes de entregar qualquer projeto com esta identidade:

- A tela principal comunica valor em até 5 segundos.
- A ação principal é óbvia.
- Há prova de execução: data, status, origem, resultado ou evidência.
- Estados de erro/vazio/carregando existem.
- A interface continua legível em mobile.
- O visual é consistente com os tokens deste pacote.
